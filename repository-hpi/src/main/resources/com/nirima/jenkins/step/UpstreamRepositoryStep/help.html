Configures upstream repository by defining an environment variable JENKINS_REPOSITORY (and Jenkins.Repository for backwards compatibility) that will be present in the Maven environment that points to the specified repository.
The best way of using this is to create a profile in the Jenkins settings.xml: 
    
        <profile>
            <id>jenkins-maven-upstream</id>
            <activation>
                <property>
                    <name>env.JENKINS_REPOSITORY</name>
                </property>
            </activation>
            <repositories>
                <repository>
                    <id>jenkins</id>
                    <url>${env.JENKINS_REPOSITORY}</url>
                    <releases>
                        <enabled>true</enabled>
                    </releases>
                    <snapshots>
                        <enabled>true</enabled>
                    </snapshots>
                </repository>
            </repositories>
            <pluginRepositories>
                <pluginRepository>
                    <id>jenkins</id>
                    <url>${env.JENKINS_REPOSITORY}</url>
                    <releases>
                        <enabled>true</enabled>
                    </releases>
                    <snapshots>
                        <enabled>true</enabled>
                    </snapshots>
                </pluginRepository>
            </pluginRepositories>
        </profile>

This profile is then automatically enabled if upstream repository is configured by executing the step.
Note: The step should be executed before any maven steps.
If the upstream job is a pipeline job it shoud use the Pipeline Maven Integration Plugin (withMaven {...} step) in order to provide maven repository that can be used as upstream repository.
